Privacy Policy

Pomodoro Timer

Last updated / Terakhir diperbarui: April 16, 2026

This Privacy Policy explains how Pomodoro Timer handles user data. Based on the current implementation of the app, the app is designed to operate primarily locally on the user's device.

1. Summary

Pomodoro Timer is a focus timer app that provides session settings, usage statistics, notification sound playback, and an option to read audio already available on an Android device if the user grants permission.

In short: the app does not require a user account, does not operate its own backend server, does not use ads, and does not send personal user data to the developer's server based on the current codebase.

2. Data Processed

The app may process the following data locally on the device:

  • App settings, such as focus duration, break duration, language, volume, and notification sound preferences.
  • Timer usage statistics, such as focus duration, break duration, completed sessions, and record dates.
  • Android device audio metadata, if permission is granted, such as song title, display name, artist, album, duration, and file reference or content URI.

3. How the Data Is Used

The data is used only for app functionality, including:

  • Saving user preferences so the app keeps the user's chosen settings.
  • Displaying timer usage statistics inside the app.
  • Reading local device audio so it can be selected or played on the user's Android device.
  • Playing built-in notification sounds.

4. Data Storage

Based on the current codebase, settings and statistics are stored locally on the user's device using local app storage. This data is not transmitted to the developer's server.

The app also does not require account registration and does not provide cloud synchronization in the current implementation.

5. Device Permissions

The app may request the following permissions on Android:

  • READ_MEDIA_AUDIO on Android 13 and above to read audio already stored on the device.
  • READ_EXTERNAL_STORAGE on Android 12 and below for the same purpose.

These permissions are functional and used only when the user wants to grant access to device audio. If the permission is not granted, the device audio feature may not work, but the core timer feature can still be used.

6. Data Sharing with Third Parties

Based on the current codebase, the app does not sell user data and does not share personal user data with the developer's server or advertising networks.

The app uses some third-party libraries for local app functions, such as preference storage and audio playback. However, in the current implementation there is no analytics, advertising, user authentication, or external backend data transmission.

7. Data Retention and Deletion

Settings and statistics remain on the user's device until the user clears app data, uninstalls the app, or uses the in-app statistics clearing feature if available.

8. Children

The app is not specifically directed to children and is not known to intentionally collect children's personal data based on the current implementation.

9. Changes to This Privacy Policy

This Privacy Policy may be updated from time to time to reflect feature changes, regulatory changes, or changes in how the app processes data. The latest version will be published on this page.

10. Contact

If you have any questions about this Privacy Policy, please contact the developer using the contact email listed on the app's Google Play listing.

Kebijakan Privasi ini menjelaskan bagaimana aplikasi Pomodoro Timer menangani data pengguna. Berdasarkan implementasi aplikasi saat ini, aplikasi ini dirancang untuk bekerja terutama secara lokal di perangkat pengguna.

1. Ringkasan

Pomodoro Timer adalah aplikasi timer fokus yang menyediakan pengaturan sesi, statistik penggunaan, pemutaran audio notifikasi, serta opsi untuk membaca daftar audio yang sudah tersedia di perangkat Android pengguna jika izin diberikan.

Intinya: aplikasi ini tidak memiliki akun pengguna, tidak memiliki server backend sendiri, tidak menggunakan iklan, dan tidak mengirim data pribadi pengguna ke server pengembang berdasarkan kode proyek saat ini.

2. Data yang Diproses

Aplikasi dapat memproses data berikut secara lokal di perangkat:

  • Pengaturan aplikasi, seperti durasi fokus, durasi istirahat, bahasa, volume, dan preferensi suara notifikasi.
  • Statistik penggunaan timer, seperti durasi fokus, durasi istirahat, jumlah sesi yang selesai, dan tanggal pencatatan statistik.
  • Metadata audio perangkat di Android jika pengguna memberi izin, seperti judul lagu, nama file tampilan, artis, album, durasi, dan referensi file atau content URI.

3. Cara Data Digunakan

Data tersebut digunakan hanya untuk fungsi aplikasi, yaitu:

  • Menyimpan preferensi pengguna agar aplikasi tetap sesuai pengaturan terakhir.
  • Menampilkan statistik penggunaan timer di dalam aplikasi.
  • Membaca audio lokal pengguna agar dapat dipilih atau diputar dari perangkat Android pengguna.
  • Memutar suara notifikasi bawaan aplikasi.

4. Penyimpanan Data

Berdasarkan kode proyek saat ini, pengaturan dan statistik disimpan secara lokal di perangkat menggunakan penyimpanan lokal aplikasi. Data tersebut tidak dikirim ke server pengembang.

Aplikasi juga tidak mewajibkan pengguna membuat akun dan tidak menyediakan fitur sinkronisasi cloud pada implementasi saat ini.

5. Izin Perangkat

Aplikasi dapat meminta izin berikut pada Android:

  • READ_MEDIA_AUDIO pada Android 13 ke atas untuk membaca daftar audio yang sudah ada di perangkat.
  • READ_EXTERNAL_STORAGE pada Android 12 ke bawah untuk tujuan yang sama.

Izin tersebut bersifat fungsional dan digunakan hanya saat pengguna ingin memberi akses ke audio perangkat. Jika izin tidak diberikan, bagian fitur audio perangkat mungkin tidak dapat digunakan, tetapi fungsi timer dasar tetap dapat berjalan.

6. Pembagian Data kepada Pihak Ketiga

Berdasarkan kode proyek saat ini, aplikasi ini tidak menjual data pengguna dan tidak membagikan data pribadi pengguna ke server pengembang atau jaringan iklan.

Aplikasi menggunakan beberapa pustaka pihak ketiga untuk fungsi lokal aplikasi, seperti penyimpanan preferensi dan pemutaran audio. Namun, dari kode aplikasi saat ini, tidak ada integrasi analytics, iklan, autentikasi pengguna, atau pengiriman data ke layanan backend pihak ketiga.

7. Retensi dan Penghapusan Data

Data pengaturan dan statistik tetap berada di perangkat pengguna sampai pengguna menghapus data aplikasi, menghapus instalasi aplikasi, atau menggunakan fitur hapus statistik bila tersedia di dalam aplikasi.

8. Anak-Anak

Aplikasi ini tidak secara khusus menargetkan anak-anak dan tidak diketahui mengumpulkan data pribadi anak-anak secara sengaja berdasarkan implementasi saat ini.

Kebijakan Privasi ini dapat diperbarui dari waktu ke waktu untuk mencerminkan perubahan fitur, perubahan regulasi, atau perubahan cara aplikasi memproses data. Versi terbaru akan dipublikasikan pada halaman ini.
